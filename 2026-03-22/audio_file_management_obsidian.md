## 結論

いちばん大事なのは、**素材と編集中と完成版を絶対に混ぜないこと**です。

管理単位は次の4層にすると安定します。

- **シリーズ**
- **回 / 案件 / エピソード**
- **収録日（セッション）**
- **テイク**

つまり、

- どのシリーズか
- 何話目か
- いつ録ったか
- 何テイク目か

が、**フォルダ名とファイル名の両方で分かる状態**が理想です。

---

## 基本ルール

### 1. RAWは不変にする

収録直後のWAVは**元データ**として扱い、以下を徹底します。

- 消さない
- 上書きしない
- 編集しない
- 必要ならバックアップを取る

つまり、ファイルは大きく次の3系統に分けます。

- **RAW**：収録直後の元データ
- **編集用**：カット・整音・DAWプロジェクト
- **完成版**：最終マスター、納品版、配信用

### 2. 1回の収録と1本の作品を分ける

1回の収録で、以下が混ざることがあります。

- 別シリーズ
- 同シリーズの別回
- リテイク
- 予備収録

そのため、**収録日だけで管理しない**のが重要です。

### 3. 1フォルダ = 1セッション or 1テイクのまとまり

今回のように、1回の収録で次の5ファイルが出る場合：

- 全マイク合体WAV × 1
- 各マイク個別WAV × 4

この5つは、**同じテイクの1セット**として扱います。

## 推奨ディレクトリ構成

```text
Audio_Archive/
├── Series_A/
│   ├── EP001/
│   │   ├── 2026-03-22_rec01/
│   │   │   ├── 01_raw/
│   │   │   │   ├── take01/
│   │   │   │   ├── take02/
│   │   │   │   └── retake01/
│   │   │   ├── 02_edit/
│   │   │   ├── 03_mix/
│   │   │   ├── 04_master/
│   │   │   └── 99_docs/
│   │   └── archive/
│   └── EP002/
│
├── Series_B/
│   └── EP001/
│
└── _common_docs/
```

### 考え方

- **シリーズ単位**で最上位を分ける
- その中を**話数 / 案件単位**で分ける
- その中を**収録日セッション単位**で分ける
- その中に**RAW / 編集 / 完成 / メモ**を置く

この構造にすると、[[#編集、アーカイブ、再利用]]に強くなります。

## 各フォルダの役割

### 01_raw

収録直後の元データ置き場です。

入れるもの：

- 合体WAV
- 各マイクWAV 4本
- 収録機から出たそのままのファイル

ルール：

- 名前変更はしてもよいが、中身は触らない
- ノイズ除去版はここに置かない
- 編集済みデータも置かない

### 02_edit

作業途中のファイル置き場です。

入れるもの：

- DAWプロジェクト
- カット編集途中
- 整音途中
- 仮処理ファイル

### 03_mix

確認用の書き出し置き場です。

入れるもの：

- 仮ミックス
- 演者確認用
- ディレクターチェック用

### 04_master

最終成果物です。

入れるもの：

- 最終マスター
- 納品版
- 配信用変換前の基準ファイル

### 99_docs

メモ類です。

入れるもの：

- 台本
- テイクリスト
- 良テイクメモ
- NGメモ
- マイク割り当て表
- 収録ログ

## ファイル名ルール

おすすめの基本形はこれです。

```text
[シリーズ]_[話数or案件]_[収録日]_[take]_[種別]
```

例：

```text
SERA_EP003_20260322_TK01_MIX.wav
SERA_EP003_20260322_TK01_MIC1_Host.wav
SERA_EP003_20260322_TK01_MIC2_GuestA.wav
SERA_EP003_20260322_TK01_MIC3_GuestB.wav
SERA_EP003_20260322_TK01_MIC4_GuestC.wav
```

### 入れるべき要素

#### 必須

- シリーズ名
- 話数 / 案件番号
- 日付
- テイク番号
- 種別

#### あると便利

- 人名
- 役名
- 役割
- バージョン
- OK / NG / BEST

例：

```text
SERA_EP003_20260322_TK02_MIC1_Masaru_OK.wav
SERA_EP003_20260322_TK02_MIX_REF.wav
SERA_EP003_20260322_TK02_EDIT_v02.wav
SERA_EP003_20260322_MASTER.wav
```

### 文字ルール

推奨：

- 英数字
- アンダースコア `_`
- 日付は `YYYYMMDD`
- テイクは `TK01` のようにゼロ埋め

避けたいもの：

- 全角記号
- 空白だらけの名前
- `final_final_fix.wav` のような曖昧名

## テイクとリテイクの扱い

ここは最初にルールを決めておくと後で楽です。

### パターンA：全部TKで通す

```text
TK01
TK02
TK03
TK04
```

長所：

- 単純
- 迷わない
- 並び替えしやすい

### パターンB：後日録り直しだけRTにする

```text
TK01
TK02
RT01
RT02
```

長所：

- 後日リテイクが一目で分かる

### おすすめ

管理を簡単にするなら、まずは**全部TK**でも十分です。  
後日録りだけ区別したいなら、**ファイル名ではなくフォルダ側で区別**するのが分かりやすいです。

例：

```text
2026-03-22_rec01/take01
2026-03-22_rec01/take02
2026-03-29_rec02/retake01
```

## 運用ルール

### 1. `final` を乱用しない

悪い例：

```text
final.wav
final2.wav
final_fix.wav
final_final.wav
```

良い例：

```text
SERA_EP003_EDIT_v01.wav
SERA_EP003_EDIT_v02.wav
SERA_EP003_MASTER_v01.wav
SERA_EP003_MASTER_v02.wav
SERA_EP003_MASTER.wav
```

### 2. マイク番号だけでなく人名も入れる

例：

```text
MIC1_Host
MIC2_GuestA
MIC3_GuestB
MIC4_GuestC
```

これにより、

- 誰のノイズか
- 誰の咳か
- 誰を録り直すべきか

が一瞬で分かります。

### 3. 収録メモを残す

`99_docs/recording_log.md` を作っておくと強いです。

入れる項目例：

- 収録日
- シリーズ名
- 話数
- 出演者
- マイク割り当て
- 良かったテイク
- NG理由
- 要リテイク箇所
- 編集指示

例：

```text
Best Take: TK03
Retake Needed: opening line only
Noise Note: MIC2 に机ノイズあり
```

### 4. シリーズごとに命名を変えない

最低限、次は統一推奨です。

- 日付：`YYYYMMDD`
- テイク：`TK01`
- マイク：`MIC1`
- 合体：`MIX`
- 完成：`MASTER`

### 5. 英数字中心にする

将来的に以下との相性が良いです。

- DAW
- NAS
- クラウド
- 他人との受け渡し
- 自動処理スクリプト

---

## 編集、アーカイブ、再利用

この構造にしておくと、次の作業が非常に楽になります。

### 編集

- どのテイクを使うか分かりやすい
- 誰のマイクか即判別できる
- 元データを壊さずに作業できる

### アーカイブ

- 後から別編集に使い回せる
- シリーズごとに圧縮・移動しやすい
- 年月単位で外付けに逃がしやすい

### 再利用

- 別作品への流用
- ボイス素材集化
- セリフ単位の抜き出し
- 再ミックス

## 最小構成の実例

迷ったら、まずはこれで十分です。

```text
Audio_Archive/
└── Series_A/
    └── EP003/
        └── 2026-03-22_rec01/
            ├── 01_raw/
            │   ├── SERA_EP003_20260322_TK01_MIX.wav
            │   ├── SERA_EP003_20260322_TK01_MIC1_Host.wav
            │   ├── SERA_EP003_20260322_TK01_MIC2_GuestA.wav
            │   ├── SERA_EP003_20260322_TK01_MIC3_GuestB.wav
            │   ├── SERA_EP003_20260322_TK01_MIC4_GuestC.wav
            │   └── ...
            ├── 02_edit/
            ├── 03_mix/
            ├── 04_master/
            └── 99_docs/
                └── recording_log.md
```

---

## 要点まとめ

- **RAWは神聖。触らない**
- **素材・編集中・完成版を分離する**
- **シリーズ → 話数 → 収録日 → テイク** で整理する
- **マイク番号だけでなく人名や役割も入れる**
- **`final` ではなく `v01` `v02` で管理する**
- **収録ログを残して、ベストテイクを明記する**
