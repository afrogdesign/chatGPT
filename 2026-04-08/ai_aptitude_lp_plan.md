## 🎯 企画の結論

このLPの本質は、**「どの学科向きか」を当てることではなく、「君の今の好きが、ここで何者に変わるか」を見せること** にあります。

そのため、企画の軸は次の通りです。

| 観点 | 要点 |
|---|---|
| LPの役割 | 興味を進路に翻訳する入口 |
| 入口の切り口 | 学科名ではなく「好きなこと」から入る |
| 結果表示 | 向いている領域・作れるもの・近い学科・将来像を見せる |
| ゴール | OC予約・資料請求・LINE登録などの軽いCVを取る |
| 成功条件 | 診断で終わらず、次の導線まで自然につなぐ |

---

## 🧭 LPの役割

AI適性診断LPは、大学説明ページではなく、**受験生の自分ごと化を生むための入口施策** として設計します。

| 役割 | 内容 |
|---|---|
| 入口施策 | 広告・SNS・学校案内から流入した人を受け止める |
| 自分ごと化 | 「この大学、自分に関係あるかも」を作る |
| 進路翻訳 | 興味を学科・制作・職業へつなぐ |
| 深掘り導線 | 作品紹介、学科ページ、先輩ナビへ送る |
| CV獲得 | OC予約・資料請求・LINE登録へ接続する |

> [!tip]
> このLPは「大学の情報を読む場所」ではなく、**自分の興味を進路へ変換する装置** と考えるとブレにくいです。

---

## 👥 想定ターゲット

メインターゲットは、**高校2年生男子の理系関心層**です。
ただし、純工学だけではなく、ゲーム・映像・建築・デザイン寄りまで拾う設計が必要です。

| ターゲット | 特徴 |
|---|---|
| 第一ターゲット | 高校2年生男子、理系関心あり、まだ志望校は固まっていない |
| 関心領域 | ゲーム、AI、ロボット、電子工作、3DCG、映像、建築、デザイン |
| 第二ターゲット | 保護者、進路に迷う高校生、作ることが好きな文理横断層 |

> [!info]
> 学科名だけでは刺さりにくいため、**「何が作れるか」「どんな未来につながるか」** を先に見せるのが重要です。

## 🖼️ ファーストビュー設計

ファーストビューの目的は、**とにかく診断を始めてもらうこと** です。

### コピー案
- **好きなことから、進路を見つけよう。**
- ゲーム、AI、ロボット、映像、建築。
- 君の興味が、大学で何に変わるかを3分で診断。

### CTA案
- **3分で診断する**
- **自分に向く分野を見る**

### 表示したい要素
- キャッチコピー
- 補足説明
- CTAボタン
- 分野横断のビジュアル
- 「7問だけ」「すぐ結果が出る」などの安心要素

## ❓ 診断設計の基本

診断は、長すぎず、迷わせず、テンポよく進める必要があります。

| 項目 | 推奨内容 |
|---|---|
| 質問数 | 6〜8問程度 |
| 形式 | 4択中心の選択式 |
| UI | スマホ前提、タップしやすい |
| 文体 | 短文、やさしい言葉 |
| 診断の方向 | 性格占いではなく、興味・適性の方向を見る |

### 質問テーマ例
- 休日にやってみたいこと
- すごいと思う技術や表現
- 作るなら何を作りたいか
- 将来に近いイメージ
- ワクワクする言葉
- 学び方の好み

> [!note]
> 難しい質問よりも、**「どれにワクワクするか」** を選ばせる方が、高2には入りやすいです。

## 🧠 診断結果タイプ案

結果タイプは、**5〜6種類程度**が最も扱いやすいです。

| タイプ名 | 向いている領域 | 訴求の方向 |
|---|---|---|
| 発明・実装タイプ | ロボット、電子、制御 | 動くものを作りたい |
| AI・システム設計タイプ | 情報、AI、プログラム | 仕組みを考えるのが好き |
| ゲーム・体験設計タイプ | ゲーム、UI、企画 | 遊びや体験を作りたい |
| ビジュアル・空間創造タイプ | 映像、建築、デザイン | 見た目や空間に強い |
| 医療・生活支援テックタイプ | 医療工学、実用技術 | 人の役に立つ技術が好き |
| 横断クリエイタータイプ | 複数分野横断 | 一つに決めず広く触りたい |

### 結果画面で見せる内容
- タイプ名
- あなたの傾向
- 向いている領域
- 近い学科
- 作れそうなもの
- 向いている仕事
- 次のおすすめ行動

## ⚙️ 診断ロジックの考え方

診断の中核は、**完全AI任せではなく、ルールベース + AI文章整形** にするのが安全です。

| 設計要素 | 内容 |
|---|---|
| スコア方式 | 各回答にタイプ別の点数を付与 |
| 判定方式 | 最も高いタイプを主診断、2位を副タイプにする |
| AIの役割 | 結果文を自然な日本語に整える |
| メリット | 結果が安定し、学科導線も制御しやすい |

> [!warning]
> 自由生成だけにすると、結果がブレやすく、大学公式施策としては危険です。

---

## 🤖 AIの使い方

このLPでのAIは、**主役ではなく翻訳役** です。

### AIにやらせること
- 回答傾向の要約
- 結果文の自然化
- 向いている理由の言語化
- タイプ別の訴求文の出し分け

### AIにやらせないこと
- 完全自由会話
- 占い的な性格診断
- 公式情報を逸脱した学科説明
- 制御不能な自由生成

> [!tip]
> AIは「すごいもの」ではなく、**受験生にとって分かりにくい情報を、自分向けに言い換える道具** として使うのが正解です。

## 🔗 学科とCVへの導線設計

診断の面白さだけで終わらせず、**学科理解とCVに接続する設計** が必要です。

### 推奨フロー

```text
広告 / SNS / 学校案内
↓
AI適性診断LP
↓
診断結果
↓
作品紹介 / 学科詳細 / 先輩ナビ
↓
OC予約 / 資料請求 / LINE登録
```

### 結果画面から出すCTA
- このタイプの作品を見る
- 向いている学科を見る
- 先輩の話を読む
- オープンキャンパスで確かめる
- 保護者向け情報を見る

> [!info]
> 最初のゴールは出願ではなく、**軽い接点をつくること** です。

## 📈 見るべきKPI

このLPでは、PVよりも **興味 → 理解 → 行動** の流れを見ます。

| KPI | 見る意味 |
|---|---|
| LP到達率 | 流入導線が機能しているか |
| 診断開始率 | ファーストビューが刺さっているか |
| 診断完了率 | 離脱せず最後まで進むか |
| 結果滞在時間 | 結果が読まれているか |
| 学科詳細遷移率 | 自分ごと化から理解へ進んだか |
| 作品閲覧率 | 「作れる未来」が見られているか |
| OC予約率 | 行動に変わっているか |
| 資料請求率 | 本人・保護者双方に刺さったか |
| 再訪率 | 単発で終わっていないか |

## 💰 予算の目安

元の整理を踏まえると、AI適性診断LPの概算は **80万〜180万円程度** が現実的です。

| プラン | 内容 | 予算感 |
|---|---|---|
| ライト | 診断 + 結果 + 学科導線 | 80万〜110万円 |
| 標準 | 上記 + AI文章整形 + 作品連携 + 計測強化 | 120万〜160万円 |
| 強化 | 上記 + LINE連携 + 結果保存 + ABテスト | 160万〜180万円 |

## 🚫 やってはいけないこと

この企画で失敗しやすいポイントは次の通りです。

| NG | 理由 |
|---|---|
| 学科説明を先に出す | 抽象的で離脱しやすい |
| 性格占いっぽくする | 進路導線に弱い |
| AI自由会話に寄せすぎる | 精度と制御が不安定 |
| 質問数を増やしすぎる | 完了率が落ちる |
| 診断で終わる | CVにつながらない |
| 「AIすごい」で終わる | 大学の魅力に接続しない |
| 工学寄りに絞りすぎる | ゲーム・映像・デザイン層を落とす |

> [!danger]
> 一番危ないのは、**面白い診断だったね** で終わることです。結果の先に、必ず学科・作品・OCへの導線を置く必要があります。

## ✅ 最終整理

| 論点 | 結論 |
|---|---|
| このLPの本質 | 「好き」を「進路」に翻訳する入口施策 |
| 一番大事な見せ方 | 学科名より先に、未来の自分像と作れるものを見せる |
| 診断設計 | 6〜8問、選択式、興味ベース |
| 実装方針 | ルールベース診断 + AI文章整形 |
| AIの役割 | 翻訳・整理・言語化 |
| 成功条件 | 結果から学科・作品・CVへ自然に流すこと |
| 最初のゴール | OC予約・資料請求・LINE登録 |

> [!abstract]
> **AI適性診断LPは、大学の説明ページではなく、受験生が「ここ、自分に合っているかも」と感じる入口をつくる施策です。**
> 
> 学科情報を並べるのではなく、**今の興味 → 学び → 作れるもの → 将来像** の流れを見せることで、初めて強い受験導線になります。
