最初に結論

  • 消したくない元データがあるなら、最初から「同期」を選ばない のが基本です。
  • バックアップ用途なら「追加コピー」または「アップデート」系が安全寄り です。

1. このソフトは何をするものか

Sync!Sync!Sync! は、指定した 2 つのフォルダを比較して、片方から片方へコピーしたり、差分だけ更新したり、双方向で内容を揃えたりするための macOS 用ユーティリティです。言い換えると、「Finder で手作業でやるファイル整理を、ルール付きで自動化するソフト」 です。

向いていること

  • 外付けSSDへの日常バックアップ
  • 作業フォルダの差分更新
  • 2台のMac間で一部フォルダの内容を揃える
  • 更新分だけコピーして時間短縮する

勘違いしやすい点

  • 「同期」= 常に安全なバックアップ、ではありません
  • 削除や上書きも同期対象になることがあります
  • モード次第で、古いファイルが消えることがあります
  • 双方向では、どちらが正なのか曖昧になりやすいです
重要： Sync 系ソフトは 「ファイルを守るソフト」ではなく、「ルールどおりに揃えるソフト」 です。したがって、ルール設定を誤ると、意図どおりに壊れます。ここが Time Machine のような「まず保護」が思想の中心にあるバックアップと大きく違うところです。

2. LE版の立ち位置と最新状況

今回の対象は Sync!Sync!Sync!LE 6.2.4 です。公式配布ページ側では 6.2.4 表記と、後継系と思われる Sync4 beta への言及 が確認できます。ただし、実際の運用現場では 「長年更新されていない定番ツールとして使われ続けている」 というニュアンスが強いです。

項目 状況 実務上の見方
現在確認できる版 6.2.4 現行利用者の目線では、事実上この版が定番
公式の新展開 Sync4 beta への導線あり 将来の主流候補だが、安定版として定着したとは言い切れない
Apple Silicon / 新しい macOS 個人レベルでは動作報告あり 公式に広く保証されている印象ではないため、環境更新前に必ずテスト推奨
LE版の性格 無料寄り・簡易版 まず使うには十分だが、全機能版との差や将来保守は意識した方がよい
実務的な理解： いまでも使えるケースは多いですが、「最新のmacOSでも永続的に安心」までは言えません。大切な運用では、OSアップデート直後に本番投入せず、まずテストフォルダで差分確認してから使うのが安全です。

3. 同期方式の全体像

このソフトがわかりにくい最大の理由は、「どちら向きに」「何を基準に」「削除も反映するのか」 がモードによって大きく違うことです。そこでまず、発想で分けます。

① 追加型

元にあるファイルを、先にない場所へ足していく考え方。既存データを壊しにくい。

② 更新型

新しいもの / 変わったものだけ置き換える考え方。効率が良いが、上書きが起こる。

③ 鏡合わせ型

片方を基準に、もう片方を同じ状態に揃える考え方。削除も反映しやすい。

④ 双方向型

両方の変更を見て、互いに反映する考え方。便利だが、競合・削除伝播の理解が必要。

代表的な理解のしかた

方式イメージ ざっくり意味 バックアップ向きか
追加コピー ないものだけ足す ◎ 安全寄り
アップデート 新しいものだけ上書き更新する ○ 条件つきで使いやすい
同期（片方向ミラー系） 元を基準に先を同じ状態へ寄せる △ バックアップというより複製・反映
双方向同期 双方の差分を統合する △〜× 慣れていないと危険

4. 方式ごとの できる / できない 一覧表

ここが一番重要です。実際の名称は表示言語やバージョン差で多少違って見えることがありますが、挙動としては次の理解でほぼ整理できます。

方式 先にないファイルをコピー 先にある古いファイルを更新 先にしかない余分なファイルを消す 元で削除したことを反映 双方向で取り込む 初心者向け安全度
追加コピー ○ × または限定的 × × × ◎
アップデート ○ ○ × × × ○
片方向同期 / ミラー ○ ○ ○ のことがある ○ のことが多い × △
双方向同期 ○ ○ 条件次第 条件次第で伝播 ○ ×

超重要な読み替え

  • 「コピーされる」こと と 「消されない」こと は別問題です。
  • 「同期」 という言葉がついている方式は、しばしば 削除まで揃える 発想を含みます。
  • バックアップ目的なら、「削除を伝播しない方式」を優先 した方が事故に強いです。
事故が起きる典型： 元フォルダで間違って大量削除 → 「同期」を実行 → バックアップ先でも同じ削除が反映。
これはソフトの誤動作ではなく、設定どおりに正しく動いている事故 です。

5. ファイル同期ルールの考え方

「なぜこのファイルがコピーされたのか / されなかったのか」を理解するには、ソフトが何を比較しているかを押さえる必要があります。一般に Sync!Sync!Sync! 系では、以下のような情報を基準に比較します。

比較要素 意味 注意点
ファイル名・階層 同じ場所に同じ名前があるか 名前を変えた場合、更新ではなく「別ファイル」と認識されやすい
更新日時 どちらが新しいかの判断材料 コピー方法によって日時が変わると、意図せず新旧判定が変わることがある
サイズ 中身が変わった可能性の判断材料 同サイズでも中身が違う場合は見抜けないことがある
削除の有無 片方から消えたものをどう扱うか 同期系ではここが一番危険

ルールを言語化するとこうなる

1同じパスに同名ファイルがあるか を見る。
2なければ、方式に応じて 新規コピーするか / しないか を決める。
3あれば、更新日時やサイズなどを見て 上書き更新するか / そのままにするか を決める。
4片方にしかないファイルについて、方式が同期系なら 削除・移動・複製の整合 を取る。
5除外ルールがある場合、対象から外す。
現場での重要点： Finder でファイル名変更や整理移動をすると、ソフトから見ると「更新」ではなく 削除 + 新規追加 に見えることがあります。このため、ミラー系や双方向系では思わぬ動きが出ることがあります。

6. 安全な使い方

初めて使う場合は、次の順で進めるのが安全です。

1本番データでいきなり試さない。 まず小さなテストフォルダを 2 つ作り、数個のファイルだけで挙動確認する。
2最初は「追加コピー」か「アップデート」から始める。 いきなり同期や双方向を選ばない。
3実行前のプレビューや差分表示があるなら必ず確認する。 とくに削除対象の有無を見る。
4バックアップ先は、元データと別ディスクにする。 同一ディスク内コピーだけでは障害対策として弱い。
5大事な運用は Time Machine 等と併用する。 Sync 系は履歴保護より状態合わせが得意。

安全度の高い運用パターン

用途 おすすめ 理由
作品データの避難コピー 追加コピー 元で消しても先が消えにくい
作業フォルダの定期更新 アップデート 新しいものだけ反映できる
配布用ミラー複製 片方向同期 元と同じ状態に揃えたい用途に合う
2台のMacで文書を往復編集 双方向同期 + 競合理解 便利だが、削除・重複・競合対応が前提

7. 用途別おすすめ設定

ケースA：写真・デザイン・原稿のバックアップ

おすすめ： 追加コピー もしくは アップデート。
理由： 間違って元データを消したとき、バックアップ先まで連動削除されにくいからです。

PSD / AI / RAW / WAV など、大きくて重要な元データでは、「同期」より「蓄積型」寄りの運用 が安全です。

ケースB：納品フォルダを常に最新状態に揃えたい

おすすめ： アップデート または 片方向同期。
元フォルダを正本にして、先を常に配布用複製にする運用です。ただし、先で独自に何か追加すると消える可能性 があるため、先は編集しない方が安全です。

ケースC：ノートや資料を2台のMacで往復したい

おすすめ： 双方向同期。ただし次が条件です。

  • 片方だけを「絶対の正」と思わない
  • 同一ファイルを両方で同時編集しない
  • 競合時にどちらが残るか、事前にテストする
  • 削除反映のルールを理解する

ケースD：外付けSSDに毎晩自動退避したい

おすすめ： アップデート + 世代バックアップ併用。
つまり、普段の差分更新は軽く回しつつ、別手段で履歴も残す と堅いです。

8. 自動実行・除外・世代管理

自動実行でできること

Sync!Sync!Sync! 系は AppleScript やスケジュール実行と組み合わせて使われることがあります。これにより、たとえば「毎晩」「マウント時」「ログイン時」の定期同期が可能です。

機能 できること 注意点
手動実行 必要時だけ実行 最も安全。確認しながら運用できる
AppleScript連携 特定設定を自動で呼び出す 設定を誤ると事故も自動化される
定期スケジュール 毎日・毎週などで自動同期 接続先ディスクの状態やマウント名ズレに注意

除外設定で重要なもの

  • キャッシュや一時ファイル
  • .DS_Store など不要な管理ファイル
  • 書き出し途中のテンポラリ
  • アプリが自動生成する索引ファイル

除外ルールを適切に設定すると、比較時間が減り、不要な差分反映も減ります。とくにメディア制作では、本体データとキャッシュを分けて考える のが重要です。

世代管理について

ここは非常に大事です。Sync!Sync!Sync! は基本的に 「現在の状態を揃える」 のが得意で、Time Machine のような 長期履歴保存が主役ではありません。したがって、次のように考えると安全です。

Sync!Sync!Sync! の得意分野

  • いま最新の状態を複製する
  • 差分だけ効率よく反映する
  • 複数場所の内容を素早く揃える

別途必要になりやすいもの

  • 古い版へ戻るための履歴
  • 誤削除から日付単位で復元する機能
  • ランサム・破損後の巻き戻し
つまり： このソフト単体で「万全のバックアップ」を期待するより、同期・複製担当 として使い、履歴担当 は Time Machine や世代バックアップに任せるのが堅いです。

9. よくある事故と対策

事故1：バックアップ先のファイルが消えた

原因： 同期 / ミラー系で削除反映が有効だった可能性。
対策： バックアップ目的では追加コピーかアップデートに切り替える。削除を伝播しない設計にする。

事故2：なぜか大量の差分が出る

原因： 更新日時のズレ、別コピーソフト経由での移動、ファイル名変更や階層移動。
対策： 一度テストフォルダで再現させ、何を差分と判定しているか確認する。

事故3：双方向で重複や競合が起きる

原因： 両方で同じファイルを編集した、または片方で名前変更・片方で更新した。
対策： 双方向対象を「同時編集しない種類のデータ」に限定する。重要ファイルは片方向運用に寄せる。

事故4：自動同期が失敗する / 別ディスク扱いになる

原因： 外付けディスクのマウント名変化、アクセス権、スリープ復帰後の未接続。
対策： 自動運用前に、対象ボリューム名とパス固定を確認する。macOS のフルディスクアクセスも見直す。

事故5：macOSでアクセスできない

近年の macOS では、デスクトップ / 書類 / 外部ボリューム / ネットワークドライブ へのアクセスに権限が必要です。同期ソフトが対象を見られない場合は、システム設定の プライバシーとセキュリティ から権限を確認してください。

鉄則： 自動化するほど、最初の設計ミスの被害は大きくなります。便利さより先に、削除・上書き・競合がどう起きるか を把握しておくべきです。

10. 最終まとめ

やりたいこと 選ぶべき方向 避けたい方向
とにかく安全に退避したい 追加コピー 削除反映のある同期
最新だけ反映したい アップデート 双方向を安易に使うこと
配布用にまったく同じ状態へ揃えたい 片方向同期 / ミラー 先で独自編集する運用
2台で相互利用したい 双方向同期 + 競合理解 重要データを無検証で双方向にすること

一言で言えば、Sync!Sync!Sync!LE は「状態を揃える道具」であって、「履歴まで守る万能バックアップ」ではありません。
なので、作品・原稿・写真・音声などの大事なデータでは、まず削除を伝播しない運用 を基本にして、必要に応じて Time Machine や世代バックアップを併用するのが最も現実的です。

おすすめの実務構成：
普段の高速差分退避 = Sync!Sync!Sync!LE（追加コピー / アップデート）
履歴と復元 = Time Machine など別系統
これがいちばん事故耐性があります。
