[!abstract] この文書について
この文書は、CODEX CLIで会話が長くなったときに、**どう整理し、どう新しい流れを作るか**を簡潔にまとめたものです。`/compact`、`/fork`、`/new`、`/resume`、`/clear` の役割と使い分けを、実運用ベースで整理しています。

## 結論
CODEX CLIでスレッドが長くなったときは、次の基準で使い分けると分かりやすいです。

- **同じ作業を続ける** → `/compact`
- **今の流れを分岐する** → `/fork`
- **完全に新しい会話にする** → `/new`
- **前の会話に戻る** → `/resume`
- **新規会話にして画面も消す** → `/clear`

> [!tip] 覚え方
> - 整理する → `/compact`
> - 枝分かれする → `/fork`
> - まっさらにする → `/new`

## 各コマンドの役割

### `/compact`
**同じ作業を続けたいが、会話が長くなってきたとき**に使います。

- いまの文脈を残したまま整理する
- 過去ログを圧縮して軽くする
- 同じ案件を続けるときの基本手段

**向いている場面**
- バグ修正を続けている
- 同じリポジトリで改善を重ねている
- 文脈は必要だが履歴が重い

### `/fork`
**今の会話をベースに別案を試したいとき**に使います。

- 現在の流れをコピーして分岐する
- 元のスレッドは残せる
- 比較検討や試作に向く

**向いている場面**
- 実装案AとBを分けて試したい
- 文章の別トーンを並行で作りたい
- 元の流れを壊さず探索したい

### `/new`
**今までの会話と切り離して、完全に新しい会話を始めたいとき**に使います。

- 文脈を引き継がない
- コンテキスト汚染を避けやすい
- 別件へ移るときに最も安全

**向いている場面**
- コード修正からデザイン相談へ移る
- 同じプロジェクトでも話題を完全に切りたい
- もう前の会話を参照したくない

### `/resume`
**以前の会話をあとから再開したいとき**に使います。

- 保存済みの会話に戻る
- 別作業のあとで再開しやすい

### `/clear`
**新しい会話を開始しつつ、画面表示も消したいとき**に使います。

- 新規会話になる
- 表示もクリアされる
- 見た目も含めて切り替えたいときに便利

> [!info] 補足
> `Ctrl+L` は画面表示を消すだけで、会話そのものは続きます。

## 使い分けの基準

### 同じ作業が長くなっただけ
→ `/compact`

**例**
- 同じ不具合を追っている
- README修正の続き
- 設計の詰めをしている

### 同じテーマで別案を試したい
→ `/fork`

**例**
- 実装案を分けて比較したい
- 別の書き方を平行して試したい
- 安全に分岐したい

### 完全に別の話へ移る
→ `/new`

**例**
- サーバーの話からSNS文案へ移る
- バグ修正から構成設計へ移る
- 文脈を完全に切りたい

## 実践ルール

> [!tip] 実運用での基本ルール
> - 重くなったが、同じ案件を続ける → `/compact`
> - 今の流れを残して別案を試す → `/fork`
> - 完全に別件へ行く → `/new`

この3つだけ覚えておけば、かなり迷いにくくなります。

## すぐ使えるコマンド集

```bash
# 同じ話を軽くして続ける
/compact

# 今の流れをコピーして別分岐
/fork

# 完全に新しい会話
/new

# 以前の会話を再開
/resume

# 新規会話 + 画面クリア
/clear
```

> [!warning] 注意
> 会話が長いからといって、必ずしも `/new` にする必要はありません。**同じ作業を続けるなら、まず `/compact`** のほうが自然です。
