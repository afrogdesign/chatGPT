1. RØDE Connect を中心にした構築図

基本思想はシンプルです。受信機を2台とも Mac に USB 接続し、RØDE Connect に入力として集約します。録音は RØDE Connect で行い、モニターは Mac 側のヘッドホン出力またはオーディオ出力先から確認します。

要点：RØDE Connect は収録の中心、RØDE Central は 機材側の事前設定 と 内部録音の管理 の役割に分けると理解しやすいです。

2. RØDE Connect の設定画面解説

Monitor Out
日本語：モニター出力先。
RØDE Connect の音をどこへ出すかを決めます。Mac内蔵スピーカー、ヘッドホン、外部出力機器などを選ぶ場所です。収録時はハウリング防止のため、基本は ヘッドホン / イヤホン 推奨です。
Monitor Mix
日本語：モニターに含める音。
Microphones Included はマイク音を含める、Microphones Excluded はマイク音を除外する設定です。収録中に自分たちの声を確認したいなら Included を使います。
Recording
日本語：録音方式。
Stereo は完成ミックスを1本で残す向き、Multi-channel は後で各トラックを分けて編集したい向きです。4人収録なら通常は Multi-channel が本命です。
Latency
日本語：遅延設定。
音の返りの速さと、Macへの負荷のバランスです。まずは Low で安定動作を優先し、返りの遅さが気になる場合だけより低遅延側を検討します。
Tooltips
日本語：説明ポップアップ。
画面上で補助説明を表示するかどうかです。慣れるまでは有効のままで問題ありません。
Optional Firmware
日本語：オプションのファームウェア。
通常更新とは別の候補を含めるか、追加的なファームウェア要素を扱うための項目に見えます。安定重視なら、むやみに変更せず、必要時のみ触るのが安全です。
Local Recordings
日本語：ローカル録音の保存先。
収録データの保存場所です。最初は既定の保存先でも構いませんが、案件ごとに探しやすくするなら専用フォルダを切ると運用が楽です。
おすすめ設定：
4人収録なら Monitor Out = ヘッドホン、Monitor Mix = Microphones Included、Recording = Multi-channel、Latency = Low から始めるのが安全です。

録音と書き出しの考え方

Multi-channel で録っておけば、書き出し時に各トラックを分離して扱いやすくなります。作業感覚としては、まず RØDE Connect 内で録り、あとから Export で DAW 向け素材を整える流れです。

  • WAV：編集向け。音質劣化なし。
  • MP3：共有向け。軽いが編集素材としては不利。
  • Frequency：サンプルレート。通常は収録全体の一貫性を優先。
  • Quality：主にMP3の品質設定。
  • Loudness (LUFs)：配信先や用途に合わせたラウドネス基準。
実務上のコツ：編集用にはまず WAV を書き出し、そのあと共有版として MP3 を別途作る流れが扱いやすいです。

モニターの考え方

収録中の確認は、基本的に Mac 側の出力を使う と理解すると整理しやすいです。受信機側の 3.5mm モニターを使う手もありますが、RØDE Connect を中心にするなら、まずは Connect の Monitor Out で統一したほうが分かりやすいです。

  • 全体を確認したい → Mac の Monitor Out にヘッドホン
  • 自分たちの声も確認したい → Monitor Mix を Microphones Included
  • ハウリングを避けたい → スピーカーではなくヘッドホン
  • 遅れが気になる → Latency を見直す

3. RØDE Central の設定解説

Audio ページで見るべきもの

Routing / Split / Merge
意味：送信機2本の音をどう出すか。
話者ごとに分けて編集したいなら Split が基本。1本のミックスとして扱いたいなら Merge です。
Safety Channel
意味：片側を低めの予備レベルで記録する考え方。
大声や不意のピークに備える保険的な設定です。現場の安全性を重視するなら有効です。
Fine Gain / Output Gain
意味：出力レベルの細かな調整。
接続先に合わせて最適レベルへ詰める項目です。録り側で余裕を持たせると事故が減ります。
Pad
意味：レベルを減衰させる。
出力が強すぎるときに落として、歪みや過入力を避けるための設定です。
HPF (High-pass Filter)
意味：低域カット。
空調、振動、服擦れ、机の振動など低い不要音を減らすのに使います。声中心なら有効な場面が多いです。
GainAssist
意味：レベルの自動補助。
Wireless PRO 系で、声量差がある話者でも使いやすくする補助機能です。現場の安定感を重視するなら有力です。

Transmitter ページで見るべきもの

Internal Recording
意味：送信機内部への録音。
特に Wireless PRO では 32-bit float 内部録音が強力です。受信側の録音に万一のトラブルがあっても、送信機側を保険にできます。
Record Mode
意味：圧縮 / 非圧縮などの録音方式。
音質優先か、収録時間優先かのバランスです。大事な収録では音質優先が基本です。
Marker / Flag
意味：収録中の印。
言い直しや重要ポイントの目印として使えます。RØDE製品では製品によってマーカーの扱いが少し異なるので、運用上はまず 収録メモ用 と考えるのが安全です。
Ø Button
意味：本体ボタンの割り当て。
ミュートや録音など、どの機能を割り当てるかを決める要素です。現場で押し間違えがない設計にしておくことが大事です。

Device ページで見るべきもの

Name / Label
意味：機器名の管理。
TX1 / TX2 の役割が混ざらないよう、収録者名や用途で整理しておくと現場がかなり楽になります。
Firmware
意味：機器のファーム更新。
相互運用性や安定動作のため、現場前に合わせておくのが基本です。
Camera Presets / Output Profiles
意味：接続先に応じたプリセット。
カメラやモバイル、PCなど接続先の違いに応じて、出力レベルや挙動を合わせるための考え方です。
Storage / Onboard Files
意味：内部録音データの管理。
送信機側の録音を消す前に、必ずバックアップを取るのが安全です。

おすすめ運用フロー

  1. まず RØDE Central で各機器のファームを更新する
  2. Routing を確認し、必要なら Split を選ぶ
  3. Wireless PRO は内部録音を有効にし、保険素材を確保する
  4. Mac に受信機2台をつなぎ、RØDE Connect へ集約する
  5. RØDE Connect は Multi-channel で録る
  6. 本番前に 10 秒テストして、全員の声とモニター返りを確認する
  7. 本番終了後、まず録音データをバックアップし、必要なら送信機内部録音も回収する
おすすめ：RØDE Connect を本収録、Wireless PRO の内部録音を保険、という二段構えにすると安心です。

4. 実践のコツ

  • 収録前に必ず10秒テスト：トラック分離、モニター、ノイズ確認。
  • 人ごとに役割を固定：GO II をゲスト、PRO をホストなど、混乱しない割り振りにする。
  • 保存先を固定：収録ごとに保存場所が変わらないようにする。
  • 送信機内部録音を保険に：特に大事な回は必須級。
  • Export後すぐバックアップ：Mac本体だけに置かない。
注意：RØDE製品はモデルやファーム差で表記が少し違うことがあります。実際の画面名が少し異なっても、考え方はほぼ同じです。
