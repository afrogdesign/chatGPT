構成の核

受信機を GO II と PRO の2台とも Mac に USB 接続し、RØDE Connect に集約します。

録音の核

Recording = Multi-channel を使い、編集向けには WAV で Export します。

モニターの核

Monitor Out = ヘッドホン、Monitor Mix = Microphones Included が基本です。

2. RØDE Connect を中心にした構築図

役割分担はこうです。RØDE Connect は本収録の中心、RØDE Central は機材側の設定と内部録音管理です。

考え方：RØDE Connect は 録る場所、RØDE Central は 機材を仕込む場所 と覚えると整理しやすいです。

3. 接続と初期設定

1

Centralで更新

最初に各機器のファームをそろえます。現場前に済ませます。

2

Routing確認

人ごとに分けたいなら Split を基本に考えます。

3

MacへUSB接続

GO II と PRO の受信機を両方 Mac へ接続します。

4

Connectで収録準備

Multi-channel とモニター設定を確認してテスト録音します。

注意：3.5mmアナログ接続は補助用途。RØDE Connect を中心にするなら、まずは USB接続優先 で考えるほうがシンプルです。

4. RØDE Connect 詳解

Monitor Out
日本語：モニター出力先。
RØDE Connect の音をどこへ出すかを決める項目です。収録用途ではスピーカーよりも ヘッドホン / イヤホン を推奨します。
Monitor Mix
日本語：モニターに含める音。
Microphones Included はマイク音を含める、Microphones Excluded はマイク音を除外する設定です。収録の確認には通常 Included が向きます。
Recording
日本語：録音方式。
Stereo は1本の完成ミックス、Multi-channel は後編集向けの分離前提です。4人収録なら通常は Multi-channel が本命です。
Latency
日本語：遅延設定。
返りの速さと安定性のバランスです。まずは Low で安定させ、必要ならより低遅延側を試します。
Tooltips
日本語：補助説明の表示。
操作に慣れるまでは有効のままで問題ありません。
Optional Firmware
日本語：オプションのファームウェア。
追加の候補や特別な更新要素を扱う項目と考えるのが自然です。安定重視なら無闇に切り替えないほうが安全です。
Local Recordings
日本語：ローカル録音の保存先。
録音ファイルをどこへ保存するかです。案件ごとに整理するなら専用フォルダを作ると後で楽です。
おすすめ値： Monitor Out = ヘッドホン / Monitor Mix = Microphones Included / Recording = Multi-channel / Latency = Low

録音と書き出し

Stereo
完成ミックスを1本で残す考え方です。素早い共有には向きますが、後編集の自由度は下がります。
Multi-channel
各トラックを後で分けて扱う前提の録音です。収録後に編集するならこちらが本線です。
WAV
編集用。非圧縮で安全です。マスター素材はこちら推奨です。
MP3
共有用。軽いですが、編集マスターには向きません。
Frequency
サンプルレート。周辺システムとそろえることが大切です。
Quality
主に MP3 の品質設定。共有用の軽さと音質のバランスを決めます。
Loudness (LUFs)
公開先に合わせてラウドネスを整えるための基準です。
実務のコツ：まず WAV で編集用素材を書き出し、その後で配布用に MP3 を作ると事故が少ないです。

イヤモニの考え方

  • 全体を確認したい → Mac の Monitor Out にヘッドホンをつなぐ
  • 自分たちの声も聞きたい → Monitor Mix を Microphones Included
  • ハウリングを避けたい → スピーカーではなくヘッドホン
  • 遅れが気になる → Latency を見直す
今の設定で起こりやすいこと： Monitor Out = 内蔵スピーカー かつ Microphones Included にすると、返り音でハウリングしやすくなります。

収録中フラッグの使い方

フラッグは、言い直し・重要ポイント・カット候補 の目印として使うのが実践的です。

  • 噛んだ場所に打つ
  • 採用したい良い発言に打つ
  • ノイズが入った場所に打つ
運用上の考え方：まずは Connect 内の編集メモ として扱うのが安全です。書き出しファイルでの扱いは、使う編集ソフト側の挙動確認を前提にしてください。

5. RØDE Central 詳解

Routing / Split / Merge
意味：送信機2本の音をどう出すか。
話者ごとに分けて編集したいなら Split が基本。まとめて扱いたいなら Merge です。
Safety Channel
意味：片側を低めレベルの保険として残す考え方。
急な大声やピーク事故に備える用途です。
Fine Gain / Output Gain
意味：出力の細かな調整。
接続先に合わせて最適なレベルへ詰めます。少し余裕を持たせるほうが安全です。
Pad
意味：レベルの減衰。
入力が強すぎるときに歪みを避けるための補助です。
HPF
意味：低域カット。
空調、振動、服擦れ、机の振動などを軽減しやすい設定です。
GainAssist
意味：声量差に対する自動補助。
話者差が大きい現場で安定感を出しやすくなります。
Internal Recording
意味：送信機内部への録音。
特に Wireless PRO は本番保険として非常に強力です。受信側トラブルの保険になります。
32-bit float
意味：後処理耐性の高い内部録音方式。
大きなレベル変化がある収録で強いです。
Record Mode
意味：圧縮 / 非圧縮などの録音方式。
音質と録音時間のバランスを決めます。大事な回は音質優先が安心です。
Marker / Flag
意味：収録中の印。
言い直しや重要箇所の目印として使います。
Ø Button
意味：本体ボタンの機能割り当て。
録音・ミュートなどの誤操作を避けるよう、現場に合わせて決めます。
Name / Label
意味：機材名の整理。
TX1 / TX2 が誰か分からなくならないよう、役割名を意識して管理します。
Firmware
意味：機器の更新。
相互運用性と安定性の基本です。本番前に合わせます。
Camera Presets / Output Profiles
意味：接続先に応じた出力の考え方。
カメラ、モバイル、PC など接続先に応じて最適化します。
Storage / Onboard Files
意味：内部録音の管理。
消す前に必ずバックアップを取るのが安全です。

実用プリセットの考え方

対談重視

Split / HPF 軽め / GainAssist 必要に応じて。あとで人ごとに整えやすくします。

事故回避重視

Safety Channel / 内部録音 / 余裕ある Gain 設定。大事な収録はこちら寄りです。

素早い共有重視

本収録は Multi-channel、共有時だけ MP3 書き出し。マスターは WAV を残します。

6. おすすめ運用フロー

  1. RØDE Central で各機器のファームを更新する
  2. Routing を確認し、必要なら Split を選ぶ
  3. Wireless PRO は内部録音を有効にして保険を確保する
  4. 受信機2台を Mac に USB 接続する
  5. RØDE Connect で Recording = Multi-channel にする
  6. Monitor Out = ヘッドホン、Monitor Mix = Microphones Included にする
  7. 10秒テスト録音し、全員の声・トラック分離・返り音を確認する
  8. 本番収録する
  9. 収録後はまず WAV を Export し、必要なら MP3 も作る
  10. 最後に Connect録音と内部録音の両方をバックアップする
理想形： RØDE Connect を本収録、Wireless PRO の内部録音を保険 にする二段構えです。

7. 最終チェックリスト

  • GO II と PRO の受信機が両方とも Mac に USB 接続されている
  • RØDE Central でファーム更新済み
  • Routing が意図どおりになっている
  • 必要なら Wireless PRO の内部録音が有効
  • RØDE Connect の Recording が Multi-channel
  • Monitor Out がヘッドホン系
  • Monitor Mix が Microphones Included
  • 10秒テスト録音済み
  • Export 後の保存先を把握している
最後に：画面上では4本見えていても、本番前の テスト録音と Export 確認 が最重要です。ここで安心を取りにいくのが正解です。
